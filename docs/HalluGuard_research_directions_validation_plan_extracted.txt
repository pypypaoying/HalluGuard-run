本文档的定位
• 不是把已有模块简单堆叠，而是把每个创新方向转化成可证伪的理论命题。
• 每个方向均包含：理论基础、核心假设、架构构想、需要验证的内容、实验设计思路。
• 最终目标是判断哪些方向有资格成为 HalluGuard 的下一代系统级创新，而不是局部 trick。
0. 执行摘要
当前 HalluGuard 的已验证价值在于：它作为 lightweight test-time correction layer，不重训 forecasting backbone，只使用 recent context 与 raw forecast，在验证集校准过的条件下执行小幅 output-level correction。公开说明中，项目强调处理 boundary jumps、trend changes、unsupported high-frequency behavior 等局部动态违规；当前最强线索包括 boundary-aware repair、selective residual smoothing、validation-calibrated routing，以及 stable guard 对 PatchTST 外部 harm 的控制。
本文档把后续研究的核心问题抽象为一个残差对齐问题：给定 recent context x、原始预测 y_hat 和真实未来 y，修正器输出 delta 后得到 y_tilde = y_hat + delta。若 residual e = y - y_hat，则 MSE 是否改善取决于修正向量与真实残差是否同向。
Delta MSE = ||delta - e||^2 - ||e||^2 = ||delta||^2 - 2 delta^T e
因此，任何创新方向都应优先证明：它能在目标不可见的测试时刻，提高 delta 与 e 的方向对齐概率，并在无法判断时 abstain。这也是本文建议把 HalluGuard 从“后处理技巧”升级为“带风险控制的 forecast output calibration framework”的理论入口。
建议优先级
• 第一优先级：Residual alignment theory + No-harm selective correction。先证明当前机制为什么有效、何时有害。
• 第二优先级：Decomposed residual dynamics corrector + Multiscale amplitude-phase support。用结构化 residual basis 扩大性能上限。
• 第三优先级：Regime-invariant correction。解决模型受限、数据集受限和外部泛化不稳。
• 探索方向：Energy/diffusion refinement 与 Time-series foundation model critic。先验证 critic separability 和 disagreement-harm correlation，再投入大模型。

1. 统一理论框架：从修正幅度转向残差方向对齐
所有 output-level correction 都可以被写成：
y_tilde = y_hat + delta(x, y_hat)
其中 delta 是修正器在测试时根据 context 和 raw forecast 产生的向量。设真实残差为 e = y - y_hat，则在平方损失下，修正有益的充要判据可以写成：
2 delta^T e > ||delta||^2
这个式子给出三个重要结论：
修正幅度不是越大越好；如果方向错，即使幅度很小也可能造成 harm。
只做 smoothing 可能在平均 MSE 上有收益，但若 smoothing 方向与真实残差不稳定对齐，就难以支撑强泛化 claim。
HalluGuard 的下一步应显式建模“何时 delta 与 e 对齐”以及“何时无法判断而应 abstain”。
1.1 建议引入的核心验证量
验证量
定义/计算方式
解释
Alignment ratio A
A = 2 delta^T e / (||delta||^2 + eps)
A > 1 时该 correction 在 MSE 意义下降低误差；A 越大，方向越可靠。

排除幅度影响后衡量方向是否正确。

Oracle action gap
loss(second-best action) - loss(best action)
衡量 action selection 是否有足够可学信号；gap 太小则 router 学不稳。
Regime stability
同一 target-free regime 中最优 action / improvement 在不同 dataset、backbone、horizon 下的一致性
验证是否存在跨域稳定的 correction law。
2. 创新方向一：动态一致性投影器
方向定义。 把 HalluGuard 从“若干手写 correction action 的选择器”升级为“把 raw forecast 投影回 context-conditioned dynamic manifold 的约束优化层”。
2.1 理论基础
该方向基于 constrained projection / trajectory manifold 的思想。raw forecast 可能在局部动态上不属于 recent context 支持的合理未来集合 M(x)：例如边界不连续、一阶差分突变、曲率异常、频谱支撑不一致。修正器不重新预测未来，而是在尽量保留 raw forecast 的前提下，找到一个更动态自洽的轨迹。
y_tilde = argmin_z ||z - y_hat||^2 + lambda * D_dyn(z, x)
其中 D_dyn 可以由 boundary、slope、curvature、spectral support、scale consistency 等约束组成。与 RevIN/NST 的差别是：RevIN/NST 主要处理输入或内部表示的非平稳性；动态一致性投影处理的是 输出轨迹是否违反 context 支持的局部动力学。
2.2 核心假设
H1-1：forecast hallucination 经常表现为少数可描述的动态约束违规，而不是任意白噪声。
H1-2：这些违规与真实 residual e 的方向存在统计相关性，因此最小扰动投影能减少误差。
H1-3：动态约束应主要作用在 forecast boundary 和近端 horizon，因为起点附近的错误具有最大传播杠杆。
2.3 架构构想
构建一个 differentiable / closed-form projection layer。它不直接用黑盒网络输出 delta，而是让小网络或验证集校准器预测约束权重 lambda，然后由显式投影求出修正。
delta* = argmin_delta ||delta||^2 + sum_k lambda_k C_k(y_hat + delta, x)
C_boundary：预测首点与 context last point + last diff 的一致性。
C_slope / C_curvature：预测头部的一阶、二阶差分与 context tail 的一致性。
C_spectral：预测频带能量与 context tail 支持范围的一致性。
C_scale：预测方差、diff std、局部 roughness 不应偏离 context 支持太多。

3. 创新方向二：带 no-harm 保证的选择性修正
方向定义。 把目标从“平均 MSE 提升”升级为“在可控 harm rate 下选择性修正”。核心不是更复杂 router，而是显式建模 correction risk 与 abstention。
3.1 理论基础
该方向受 selective prediction、reject option、conformal prediction、conformal risk control 启发。对 HalluGuard 来说，关键不是每个样本都修，而是在无法判断 correction 是否有益时保留 raw forecast。
y_tilde = action(y_hat, x), if R(x, y_hat, action) <= tau; otherwise y_hat
这里 R 不是预测误差本身，而是 correction risk score，目标是控制 P(Delta MSE > epsilon | correct)。
3.2 核心假设
H2-1：HalluGuard 的主要风险不是找不到有用 correction，而是无法可靠地区分“该修”和“不该修”。
H2-2：harm 不是随机发生，能被 target-free 特征预测，例如 stable forecast score、router margin、diff std ratio、unsupported correction magnitude、OOD context score。
H2-3：存在一个 coverage 区间，使得修正覆盖率下降不多，但 harm rate 和 worst-group harm 明显降低。
3.3 架构构想
学习或校准一个 risk score R，而非只学习 action probability。
用 validation split 选择 threshold tau，优化 risk-coverage trade-off，而非只优化平均 MSE。
输出不仅包括 corrected forecast，还包括 action、risk、coverage bucket、abstain reason。
target:  minimize mean loss under constraint  P(harm | correct) <= alpha

4. 创新方向三：结构化残差动力学建模
方向定义。 不再只根据局部异常做小幅修复，而是显式学习 forecast residual process 的结构：level、slope、trend、seasonal、horizon-dependent bias、local spike。
4.1 理论基础
该方向的核心假设是：现代 forecasting backbone 的 error 并非全是白噪声，而是存在可学习、可分解、跨模型部分共享的 residual dynamics。2026 年的 UEC-STD 预印本也沿着 architecture-agnostic error correction 的路线，提出对预测误差进行 trend/seasonal decomposition 并在不重训 forecaster 的情况下修正。
e = y - y_hat = e_level + e_slope + e_trend + e_seasonal + e_hf + e_regime
4.2 核心假设
H3-1：oracle residual 在低维结构化 basis 上有较高 explained variance。
H3-2：残差的部分 basis 权重可以由 x 与 y_hat 的 target-free 特征预测。
H3-3：decomposed residual corrector 比直接 MLP residual predictor 更稳，因为它限制了 correction 的自由度。
4.3 架构构想
用基函数组合预测 residual，而不是直接生成完整未来轨迹：
e_hat = B_boundary w_b + B_slope w_s + B_trend w_t + B_seasonal w_p + B_local w_l
B_boundary：指数衰减的边界修复基。
B_slope / B_curvature：一阶、二阶导数衔接基。
B_trend：低频多项式 / spline / DCT 低频基。
B_seasonal：周期基或 learnable Fourier basis。
B_local：稀疏局部 spike / unsupported residual basis。
最终 correction 应带 shrinkage：y_tilde = y_hat + gamma * e_hat，其中 gamma 由风险校准或 conformal-style coverage 控制。

5. 创新方向四：能量式 / 扩散式轨迹精修
方向定义。 把 raw forecast 视为条件轨迹分布附近的有偏样本，通过 energy / score 将它推回更高概率的 future-like trajectory，而不是一次性 deterministic offset。
5.1 理论基础
扩散模型与 energy-based model 的启发是：先学习 p(y | x) 或一个等价的 trajectory plausibility score，再对 raw forecast 执行 refinement。对 HalluGuard，更轻量的版本不是直接做生成式 forecasting，而是训练一个 conditional critic E_theta(x, z)：真实未来低能量，坏 forecast / stress-corrupted future 高能量。
y_tilde = y_hat - eta * grad_z E_theta(x, z)|_{z = y_hat}
5.2 核心假设
H4-1：真实未来 y、raw forecast y_hat、synthetic hallucinated forecast z_bad 在 trajectory plausibility 上可区分。
H4-2：critic 的梯度方向与真实 residual e 有统计对齐，而不只是识别“假样本”。
H4-3：synthetic stress suite 可以作为负样本生成器，而不仅是评测集。
5.3 架构构想
正样本：真实未来 y。
负样本：raw forecast、边界跳变污染、相位错位、高频注入、variance shift、delayed level shift 等 corrupted future。
训练目标：ranking loss / contrastive loss，使 E(x, y) < E(x, z_bad)。
推理时只做 1-3 步小梯度 refinement，并由 no-harm risk score 决定是否启用。

6. 创新方向五：时间序列 foundation model 作为轨迹先验/批判器
方向定义。 不让 TS foundation model 替代 backbone，而是作为外部 trajectory prior / forecast critic，用于判断 raw forecast 是否偏离跨域时间序列先验。
6.1 理论基础
Chronos、TimesFM、Moirai-MoE 等工作说明，大规模跨域预训练可以学习一定的通用时间序列轨迹先验。但 HalluGuard 不应假设 foundation model 永远更准；更合理的定位是：foundation model 提供一个外部参照，用于发现 raw forecast 的异常、OOD 或不确定区域。
6.2 核心假设
H5-1：当 raw backbone forecast 出现 hallucination 时，它与 TSFM forecast / likelihood / quantile band 的 disagreement 会升高。
H5-2：TSFM 作为 critic 比作为 teacher 更安全；不直接把 forecast 拉向 TSFM，而是用 disagreement 判断是否应修、修多少。
H5-3：TSFM prior 能缓解 HalluGuard calibration set 太小导致的数据集受限。
6.3 架构构想
signals = Disagreement(y_hat_backbone, y_hat_TSFm; amplitude, slope, spectrum, quantile, likelihood)
动作一：keep backbone，若 TSFM disagreement 低且 HalluGuard risk 低。
动作二：small correction toward TSFM prior，仅在 TSFM 历史验证上可靠且 HalluGuard risk 高时。
动作三：abstain / conservative guard，若 backbone 与 TSFM 都不确定或互相冲突。

7. 创新方向六：多尺度 amplitude-phase 支持校准
方向定义。 从粗糙的 high-frequency suppression / median smoothing，升级为多尺度、分频带、显式 amplitude-phase 的支持校准：只修正 context 不支持的尺度成分，而不是全局平滑。
7.1 理论基础
TimeMixer 从多尺度角度建模时间序列的微观/宏观模式；TimeAPN 显式讨论 amplitude 和 phase non-stationarity。这对 HalluGuard 的启发是：许多 forecast failure 并非“太抖”这么简单，而是某些频带的 amplitude、phase 或局部尺度支撑出错。
7.2 核心假设
H6-1：当前 smoothing 的收益来自删除 unsupported components，但 harm 来自误删真实动态成分。
H6-2：forecast error 在尺度/频带上具有稀疏性，即不是所有尺度都错。
H6-3：context-forecast 的 scale-wise mismatch 能预测对应尺度 residual。
7.3 架构构想
x -> {x^(s)},   y_hat -> {y_hat^(s)},   y_tilde = sum_s edit_s(y_hat^(s) | x^(s))
用 wavelet / DCT / moving average pyramid / STL-like decomposition 得到多尺度成分。
每个尺度独立计算 support score：amplitude ratio、phase shift、spectral mass、local roughness。
只对 unsupported scale 做 shrinkage 或 phase correction；对 supported scale 保持不动。

8. 创新方向七：regime-invariant correction
方向定义。 不在 dataset/backbone 层面学习 correction，而是在 target-free latent regime 层面学习 correction law。目标是让 HalluGuard 跨模型、跨数据集、跨 horizon 泛化。
8.1 理论基础
Dish-TS 将 time-series distribution shift 分成 intra-space shift 和 inter-space shift；Moirai-MoE 也指出频率标签不足以描述时间序列异质性。HalluGuard 的修正规则若以 dataset/backbone 为单位学习，很容易泛化失败；若以机制型 regime 为单位学习，则有机会形成可迁移的 correction law。
8.2 核心假设
H7-1：不同 dataset/backbone 下存在共享的 forecast failure regimes，例如 stable-low-noise、boundary-discontinuity、variance-shift、slope-break、unsupported-oscillation、delayed-level-shift。
H7-2：同一 regime 内的最优 correction action 与 improvement distribution 比按 dataset/backbone 分组更稳定。
H7-3：external harm 集中在少数可识别 regime 中，因此可以用 regime-aware abstention 控制。
8.3 架构构想
r = q_theta(x, y_hat),   delta = C_r(x, y_hat),   with regime-invariance penalty
这里的 q_theta 不是普通 MoE router，而是受机制监督或机制约束的 regime encoder。可以利用 synthetic stress labels、residual shape labels、scale mismatch labels、harm labels 对 regime 施加解释性约束。

10.5 成功与失败判据
建议的 Go 判据
• 至少一个方向在 leave-one-backbone 或 leave-one-dataset 中保持负 MSE delta，同时 harm rate 不高于当前 stable guard。
• 至少一个理论指标能解释收益：例如 alignment A、risk score、residual basis explained variance、scale mismatch correlation。
• 主机制必须同时击败 random trigger、matched smoothing、shuffled-feature router 和 dominant-action baseline。
• 外部泛化 claim 只能在更大 external benchmark 上成立；现有 external fixture 仍应定位为 harm diagnostic。

11. 面向论文叙事的建议
如果上述验证成立，论文叙事可以从“post-processing improves forecasts”升级为以下更强命题：
主命题草案
• Forecast correction should be treated as selective residual alignment under target-free risk control.
• A correction is useful only when its direction is aligned with the latent forecast residual; therefore, output guards must jointly model correction direction and abstention risk.
• HalluGuard provides a model-agnostic framework that corrects locally inconsistent forecasts when alignment can be certified and abstains when no-harm cannot be supported.
对应中文表述：HalluGuard 不是泛泛地“平滑预测”，而是在测试时使用 context 与 raw forecast 的目标不可见特征，判断 forecast 是否违反局部/多尺度/机制型动态约束；只有当修正方向有较高残差对齐可能性且 harm risk 受控时才修正。
